# Relay Deployment Guide

Complete guide for deploying an Atlax relay on any supported platform.

---

## Overview

The relay is the public-facing component of an Atlax tunnel. It:
- Listens for agent connections over mTLS 1.3 on port 8443
- Accepts client TCP connections on customer-allocated ports
- Routes traffic through multiplexed streams to the correct agent
- Exposes Prometheus metrics and health endpoints on the admin port

```
Internet clients
    │
    ▼
┌─────────────────────────────┐
│         RELAY (VPS)         │
│                             │
│  :8443   ◄── Agent mTLS    │
│  :9090   ◄── Admin/metrics  │
│  :18080  ◄── Customer port  │
│  :18070  ◄── Customer port  │
│  ...                        │
└─────────────────────────────┘
    │
    ▼ (mTLS tunnel)
Agent behind CGNAT
```

## Interactive Setup

```bash
ats setup relay
```

The wizard handles everything. For operators who prefer step-by-step control, the sections below explain each phase.

## Manual Step-by-Step

### 1. System Preparation

**Create system group (controls file access):**

```bash
# Linux
sudo groupadd --system atlax

# FreeBSD
sudo pw groupadd atlax
```

**Create service user (optional, runs the daemon):**

```bash
# Linux (systemd)
sudo useradd --system --no-create-home --shell /usr/sbin/nologin -g atlax atlax

# FreeBSD
sudo pw useradd atlax -s /usr/sbin/nologin -d /nonexistent -g atlax

# macOS (for dev/testing — launchd runs as root by default)
sudo dscl . -create /Users/atlax IsHidden 1
```

**Create directories:**

```bash
sudo mkdir -p /etc/atlax/certs
sudo mkdir -p /var/log/atlax
sudo chgrp -R atlax /etc/atlax /var/log/atlax
sudo chmod -R g+r /etc/atlax
sudo find /etc/atlax -name "*.key" -exec chmod 640 {} \;
```

### 2. Binary Installation

**Option A: Download prebuilt**

```bash
VERSION="latest"
ARCH=$(uname -m | sed 's/x86_64/amd64/; s/aarch64/arm64/')
OS=$(uname -s | tr '[:upper:]' '[:lower:]')

curl -fsSL -o /tmp/atlax-relay \
  "https://github.com/atlasshare/atlax/releases/download/${VERSION}/atlax-relay-${OS}-${ARCH}"

sudo install -m 755 /tmp/atlax-relay /usr/local/bin/atlax-relay
```

**Option B: Build from source**

```bash
git clone https://github.com/atlasshare/atlax.git
cd atlax
make build-relay
sudo install -m 755 bin/atlax-relay /usr/local/bin/atlax-relay
```

**Option C: Copy from local path**

```bash
sudo install -m 755 /path/to/atlax-relay /usr/local/bin/atlax-relay
```

### 3. Certificate Setup

Copy certs generated by `ats certs init` to the relay:

```bash
sudo cp relay.crt relay.key root-ca.crt customer-ca.crt /etc/atlax/certs/
sudo chmod 600 /etc/atlax/certs/relay.key
sudo chown -R atlax:atlax /etc/atlax/certs/
```

Required files on the relay:

| File | Purpose |
|------|---------|
| `relay.crt` | Relay TLS certificate (presented to agents) |
| `relay.key` | Relay private key |
| `root-ca.crt` | Root CA for chain validation |
| `customer-ca.crt` | Customer CA (verifies agent client certs) |

### 4. Configuration

Create `/etc/atlax/relay.yaml`:

```yaml
server:
  listen_addr: "0.0.0.0:8443"
  admin_addr: "127.0.0.1:9090"
  max_agents: 100
  max_streams_per_agent: 100
  idle_timeout: 300s
  shutdown_grace_period: 30s

tls:
  cert_file: /etc/atlax/certs/relay.crt
  key_file: /etc/atlax/certs/relay.key
  ca_file: /etc/atlax/certs/root-ca.crt
  client_ca_file: /etc/atlax/certs/customer-ca.crt

customers:
  - id: customer-dev-001
    max_connections: 1
    max_streams: 100
    ports:
      - port: 18080
        service: http
        listen_addr: "127.0.0.1"
        description: "Web service"

logging:
  level: info
  format: json

metrics:
  enabled: true
  path: /metrics
  prefix: atlax
```

**Configuration reference:**

| Field | Default | Description |
|-------|---------|-------------|
| `server.listen_addr` | — | Agent mTLS listener (host:port) |
| `server.admin_addr` | `127.0.0.1:9090` | Admin/metrics endpoint |
| `server.max_agents` | `1000` | Max concurrent agent connections |
| `server.max_streams_per_agent` | `100` | Max streams per agent |
| `server.idle_timeout` | `300s` | Idle connection timeout |
| `server.shutdown_grace_period` | `30s` | GOAWAY drain period |
| `customers[].id` | — | Must match agent cert CN |
| `customers[].ports[].port` | — | TCP port to listen on |
| `customers[].ports[].service` | — | Service name (sent in STREAM_OPEN) |
| `customers[].ports[].listen_addr` | `0.0.0.0` | Bind address (`127.0.0.1` for reverse proxy) |

### 5. Firewall

**UFW (Ubuntu/Debian/Arch):**

```bash
sudo ufw default deny incoming
sudo ufw default allow outgoing
sudo ufw allow 22/tcp comment 'SSH'
sudo ufw allow 80/tcp comment 'HTTP (ACME)'
sudo ufw allow 443/tcp comment 'HTTPS (Caddy)'
sudo ufw allow 8443/tcp comment 'Atlax agent mTLS'
# Customer ports bound to 127.0.0.1 don't need UFW rules
sudo ufw --force enable
```

**firewalld (RHEL/Fedora):**

```bash
sudo firewall-cmd --permanent --add-port=22/tcp
sudo firewall-cmd --permanent --add-port=80/tcp
sudo firewall-cmd --permanent --add-port=443/tcp
sudo firewall-cmd --permanent --add-port=8443/tcp
sudo firewall-cmd --reload
```

**AWS Security Groups:**

```bash
aws ec2 authorize-security-group-ingress \
  --group-id sg-XXXXXXXX \
  --protocol tcp --port 8443 --cidr 0.0.0.0/0 \
  --tag-specifications 'ResourceType=security-group-rule,Tags=[{Key=Name,Value=Atlax Agent mTLS}]'

aws ec2 authorize-security-group-ingress \
  --group-id sg-XXXXXXXX \
  --protocol tcp --port 443 --cidr 0.0.0.0/0 \
  --tag-specifications 'ResourceType=security-group-rule,Tags=[{Key=Name,Value=HTTPS}]'
```

### 6. System Service

**systemd (Linux):**

```ini
# /etc/systemd/system/atlax-relay.service
[Unit]
Description=Atlax Relay
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
User=atlax
Group=atlax
ExecStart=/usr/local/bin/atlax-relay -config /etc/atlax/relay.yaml
Restart=on-failure
RestartSec=5s
LimitNOFILE=65536

NoNewPrivileges=yes
ProtectSystem=strict
ProtectHome=yes
PrivateTmp=yes
MemoryDenyWriteExecute=yes
LockPersonality=yes
ReadOnlyPaths=/etc/atlax
AmbientCapabilities=CAP_NET_BIND_SERVICE
CapabilityBoundingSet=CAP_NET_BIND_SERVICE

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now atlax-relay
```

**launchd (macOS):**

```xml
<!-- /Library/LaunchDaemons/io.atlax.relay.plist -->
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>Label</key>
    <string>io.atlax.relay</string>
    <key>ProgramArguments</key>
    <array>
        <string>/usr/local/bin/atlax-relay</string>
        <string>-config</string>
        <string>/usr/local/etc/atlax/relay.yaml</string>
    </array>
    <key>RunAtLoad</key>
    <true/>
    <key>KeepAlive</key>
    <true/>
    <key>StandardErrorPath</key>
    <string>/usr/local/var/log/atlax/relay.err.log</string>
    <key>StandardOutPath</key>
    <string>/usr/local/var/log/atlax/relay.out.log</string>
</dict>
</plist>
```

```bash
sudo launchctl load /Library/LaunchDaemons/io.atlax.relay.plist
```

**rc.d (FreeBSD):**

```sh
#!/bin/sh
# /usr/local/etc/rc.d/atlax_relay

. /etc/rc.subr

name="atlax_relay"
rcvar="atlax_relay_enable"
command="/usr/local/bin/atlax-relay"
command_args="-config /usr/local/etc/atlax/relay.yaml"
pidfile="/var/run/${name}.pid"
command_background="YES"

load_rc_config $name
run_rc_command "$1"
```

```bash
sudo sysrc atlax_relay_enable=YES
sudo service atlax_relay start
```

### 7. Verification

```bash
# Service running
systemctl status atlax-relay

# Health endpoint
curl -s http://localhost:9090/healthz
# → 200 OK

# Readiness
curl -s http://localhost:9090/readyz
# → 200 OK

# Metrics
curl -s http://localhost:9090/metrics | grep atlax_relay_agents

# Full diagnostics
ats health
```

## Adding HTTPS with Caddy

If you want TLS termination in front of customer ports:

```bash
ats service add
# Answer "yes" when asked about Caddy integration
```

Or manually add to `/etc/caddy/Caddyfile`:

```
app.example.com {
    reverse_proxy localhost:18080
    encode gzip zstd
    header {
        Strict-Transport-Security "max-age=31536000; includeSubDomains; preload"
        X-Content-Type-Options "nosniff"
        X-Frame-Options "SAMEORIGIN"
        Referrer-Policy "strict-origin-when-cross-origin"
    }
}
```

```bash
sudo systemctl reload caddy
```

## Troubleshooting

| Symptom | Cause | Fix |
|---------|-------|-----|
| Agent can't connect | Port 8443 blocked | Check firewall, security group |
| TLS handshake fails | Cert mismatch | Verify agent cert is signed by customer-ca |
| Customer port not listening | Agent not connected | Check agent logs, relay logs |
| High memory usage | Too many idle streams | Reduce `idle_timeout` |
| Metrics endpoint unreachable | Admin addr bound to wrong interface | Check `admin_addr` in config |
